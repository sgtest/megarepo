224â// StrictSlash defines the trailing slash behavior for new routes. The initial
225â// value is false.
226â//
227â// When true, if the route path is "/path/", accessing "/path" will perform a redirect
228â// to the former and vice versa. In other words, your application will always
229â// see the path as specified in the route.
230â//
231â// When false, if the route path is "/path", accessing "/path/" will not match
232â// this route and vice versa.
233â//
234â// The re-direct is a HTTP 301 (Moved Permanently). Note that when this is set for
235â// routes with a non-idempotent method (e.g. POST, PUT), the subsequent re-directed
236â// request will be made as a GET by most clients. Use middleware or client settings
237â// to modify this behaviour as needed.
238â//
239â// Special case: when a route sets a path prefix using the PathPrefix() method,
240â// strict slash is ignored for that route because the redirect behavior can't
241â// be determined from a prefix alone. However, any subrouters created from that
242â// route inherit the original StrictSlash setting.
243âfunc (r *Router) StrictSlash(value bool) *Router {
244â r.strictSlash = value
245â return r
246â}
